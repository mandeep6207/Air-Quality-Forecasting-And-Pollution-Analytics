# Contributing

Thanks for your interest in contributing to this project! A few simple guidelines:

- Open an issue to discuss large changes before working on them.
- Keep changes small and focused; one feature or bug per PR.
- Run the pipeline locally (`python run_pipeline.py`) and include any updated
  outputs or metrics in `reports/` when relevant.
- Follow the coding style used in `src/` (concise functions, clear logging).
- Add tests for new functionality where practical.

Pull requests are welcome — maintainers will review and provide feedback.
