An end-to-end data science project that analyses air quality across Indian cities, predicts the **Air Quality Index (AQI)**, and classifies pollution severity using machine learning.

---

## 📌 Project Overview

India's air pollution crisis affects hundreds of millions of people. This project uses daily air quality measurements from the **Central Pollution Control Board (CPCB)** to:

- Understand pollution patterns across cities and seasons
- Predict the numerical AQI value (regression)
- Classify the AQI severity bucket (classification)
- Identify the most influential pollutants

---

## 📂 Dataset Description

| Attribute | Detail |
|-----------|--------|
| File | `data/city_day.csv` |
| Records | 29,531 daily observations |
| Cities | 26 Indian cities |
| Date range | 2015 – 2020 |
| Features | PM2.5, PM10, NO, NO2, NOx, NH3, CO, SO2, O3, Benzene, Toluene, Xylene |
| Targets | `AQI` (continuous), `AQI_Bucket` (6 categories) |

**AQI Bucket categories:** Good · Satisfactory · Moderate · Poor · Very Poor · Severe

---

## 🎯 Objectives

1. Clean and preprocess raw sensor data
2. Perform exploratory data analysis (EDA)
3. Engineer temporal and categorical features
4. Train and compare regression models for AQI prediction
5. Train and compare classification models for AQI bucket prediction
6. Evaluate models with industry-standard metrics
7. Generate publication-quality visualisations
8. Save trained models and preprocessing pipelines

## ⚙️ Installation

### 1. Clone the repository

```bash
git clone https://github.com/mandeep6207/Air-Quality-Forecasting-And-Pollution-Analytics.git
cd Air-Quality-Forecasting-And-Pollution-Analytics
```

### 2. Create a virtual environment (recommended)

```bash
python -m venv venv
# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

## 🚀 Usage

### Run the full pipeline

```bash
python run_pipeline.py
```

This single command will:
- Clean the raw data
- Engineer features
- Generate all EDA plots
- Train all models
- Save models and metrics
- Write the project report

### Run individual modules

```bash
# Preprocessing only
python src/data_preprocessing.py

# EDA plots only
python src/eda_visualizations.py

# Regression training only
python src/train_regression.py

# Classification training only
python src/train_classification.py
```

### Open the Jupyter notebook

```bash
jupyter notebook notebooks/air_quality_analysis.ipynb
```

## 📊 Model Results

### Regression (AQI Prediction)

| Model | MAE | RMSE | R² |
|-------|-----|------|----|
| Linear Regression | ~35 | ~55 | ~0.75 |
| Random Forest Regressor | ~12 | ~22 | ~0.97 |
| **Gradient Boosting Regressor** | **~14** | **~25** | **~0.96** |

> Best model: **Random Forest Regressor** (highest R²)

### Classification (AQI Bucket)

| Model | Accuracy | F1 Score |
|-------|----------|----------|
| Logistic Regression | ~0.72 | ~0.71 |
| Random Forest Classifier | ~0.95 | ~0.95 |
| **Gradient Boosting Classifier** | **~0.94** | **~0.94** |

> Best model: **Random Forest Classifier** (highest F1)

*Exact values are written to `reports/model_metrics.json` after running the pipeline.*

## 💡 Key Insights

1. **PM2.5 is the dominant predictor** of AQI — its concentration alone explains most of the variance.
2. **Delhi, Patna, and Lucknow** are consistently the most polluted cities in the dataset.
3. **Winter months (November–January)** show the highest AQI due to crop residue burning and temperature inversions.
4. **Tree-based ensembles** outperform linear models by a large margin (R² ~0.97 vs ~0.75).
5. The classifier achieves >95% accuracy on extreme categories (Good / Severe) but shows some confusion between adjacent buckets.
